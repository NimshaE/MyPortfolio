Project Overview

As an Associate Software Engineer specializing in UI/UX, I worked on the FigaBooks project, which involved designing and enhancing both user and admin platforms. The project spanned several phases, including logo redesign, feature additions, branding guideline checks, and mobile development for a time-tracking system.

Logo Redesign

The project began with recreating the company logo to a flat design suitable for both the FigaBooks user and admin platforms. This set the visual tone for the entire project.

Initial Design and Scope of Worky

I started by reviewing the existing designs before diving into new features and tabs. Our team discussed the company requirements to understand the Scope of Work (SOW) clearly. The first major task was designing the Quotation tab, ensuring it maintained consistency with the existing invoices page.

Feature Enhancements and New Tabs

To align with client requirements, I designed several new features and updated existing ones:

  • GL Codes Selection: Added to the Chart of Account Expenses.
  • Reports: Created Financial Report and GST/VAT report.
  • Settings: Updated with four user types.
  • New Fields: Added to the Create Account page, including BIC, business type, GST registration, and more.
  • User Management: Added fields like company address, logo, and user types.
  • Dashboard Enhancements: Introduced new matrices, Current ratio, and P/E graphs.
  • Sales Invoices: Added GL codes, tax, and UOM input fields.
  • Transactions Tab: Updated with an Upload Statement page.
  • Reports: Added Provisional Tax Returns and GST Audit report.

Branding Guidelines and Development Feedback

I checked the staging version with developers to ensure adherence to branding guidelines and provided feedback for adjustments.

Digital Presentation Portal Updates

SOW-2: Package Pricing and Stripe Checkout

In the second scope, I focused on adding details for Package pricing and Stripe checkout for payments. Given Stripe's built-in checkout pages, I designed mock-ups for the pricing page.

SOW-3: User Packages and Further Enhancements

The third scope revolved around FigaBooks User Packages. Key tasks included:

  • Mock-Up Designs: Payment Settings, Account Closure banner, Lockout mail, Account Retention mail.
  • Updates: Transactions, Settings, Profile, Migration Settings, Journal report, minor PDF updates.
  • User Role Enhancements: Accountant's workpaper module, Fixed Asset register, Manual Journals, Year-End Reports, and IRD report.
  • General Changes: Invoices, Quotes, Customer, Supplier module, Bank statement upload.
  • Admin Portal Enhancements: Customer and Business Subscription, Coupon module, email templates for new user, invoice/quote, verify user, and account delete.
  • After each design phase, I demonstrated the updates to the client, incorporated their feedback, and then proceeded to development.

Post-Development Review and Support Centre

After development, I reviewed the staging version against mock-up designs to ensure accuracy. The client later requested a Support Centre to assist users with a video library. I designed it as a hyperlink on the top nav bar, with each section featuring an information icon linked to relevant video tutorials.

Final Updates and Demonstrations

The final tasks included redesigning the Advisor Login (New Digital SOA request) and the Presentation Upload page of the Statement of Advisor portal, incorporating a history of previously completed digital SOA requests. These designs were updated according to feedback received during demonstration sessions.

FigaBooks - Time Tracking (Mobile Development)

Initial Client Requirements

The client required a time-tracking and attendance system for athletes. This involved creating two sections: a desktop version for the Employer portal (Sports team) and a mobile version for Coaches and Athletes.

System Features

The system needed to:

  • Generate Reports: Daily, weekly, and yearly attendance.
  • Create Accounts: Coaches could create accounts for athletes.
  • Track Attendance: Athletes could log practice start time, end time, and break time via timesheets.
  • Approval Process: Admins and coaches could confirm the information submitted by athletes.

Wireframes and Mock-Ups

I started by creating wireframes for both desktop and mobile versions. The mock-ups adhered to the existing FigaBooks design, ensuring consistency. These included:

  • Time Tracking: Timesheets and attendance reports.
  • Workspace Management: Owners could create different workspaces for various sports.
  • User Roles: Athletes as users, coaches handling the workspace, and heads of sports managing master accounts. Feedback and Finalization

After internal demonstrations and updates based on feedback, the design was finalized and presented to the client for further feedback.

The FigaBooks project showcased my ability to manage complex UI/UX design tasks, collaborate with development teams, and respond to client needs effectively. This project enhanced my skills in designing consistent, user-friendly interfaces across multiple platforms and functionalities.
